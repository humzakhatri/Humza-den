title: How I Stopped Things From Falling Through the Cracks
date: 2026-06-01
tags: [ai, workflow, sprint, team]
I am not naturally a process person. Our company moves fast and does not have much structure around how work gets planned — you figure it out as you go. For a long time, my system was a rough list and a lot of things kept in my head. It worked until I was managing a team.
When you are just shipping yourself, a rough list is fine. When you are coordinating five or six people across overlapping pieces of work, things start falling through the cracks. Someone gets assigned a task that depends on something not done yet. A design detail that was clear in my head never made it to the person building it. A blocker sits unresolved for two days because nobody flagged it explicitly.
The work was getting done, but not cleanly. Things were underdesigned when they reached the team. Dependencies were implicit. I was the bottleneck and the gap at the same time.
What changed was bringing a coding agent into the sprint — not to write code, but to help me think through and structure work before it reached anyone. I would describe what needed to be built, and the agent would push back: what does this depend on? what is already done that this can build on? what does the developer need to know before they start?
That friction — being asked to answer those questions before assigning anything — was the thing I had been skipping. Not because I did not care, but because when you are moving fast it feels like overhead. The agent made it feel like a conversation instead of paperwork.
Dependencies stopped being implicit. If task B needed task A to be done first, that was written down. If something was underdesigned, we would work through it before it went to the team. If a task was too vague to act on, it got rewritten until it was not.
Nothing here is a new idea. Dependencies, clear task definitions, tracking blockers — this is basic project management. What changed is that I actually did it, because the cost of doing it dropped low enough that it was no longer something I would skip.
That is the part I did not expect. The agent did not give me a better system. It just made the system I already knew I should have cheap enough to run.
